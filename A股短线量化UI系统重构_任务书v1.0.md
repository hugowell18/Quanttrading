# QuantPulse 交易终端 — 前端 UI 重构任务书 v1.0

> **背景说明：** > 系统后台的量化引擎（基于 Node.js + 本地 CSV 的短线均值回归与行业轮动模型 v3.0）已全面完工。
> **核心重构目标：** 废弃现有的实验性多 Tab 布局，全面对标“同花顺/东方财富”的专业量化终端，打造**“大盘全景联动 + 个股深度复盘”**的双屏主架构。

---

## 🏗️ 总体架构与技术栈要求

- **页面拓扑**：将路由或顶层导航收敛为两个核心看板：
  1. `Tab 1`：大盘全景与热点雷达 (Market Dashboard)
  2. `Tab 2`：个股量化深度复盘 (Stock Analyzer)
- **状态管理**：引入全局状态（建议 `React Context` 或 `Zustand`），管理 `selectedDate`（当前选中的复盘日期）和 `selectedStock`（当前聚焦的个股代码），实现跨 Tab 的数据联动。
- **UI 组件库**：深度复用并严格遵循 `shadcn/ui` 规范（高密度、现代化、暗黑风格兼容）。
- **图表引擎**：升级 K 线图组件（推荐原生接入 `lightweight-charts`），需支持复杂的 Marker（标记）绘制与多图表时间轴同步。

---

## 🖥️ 模块一：Tab 1 - 大盘全景与热点雷达

**设计理念**：实现“自上而下 (Top-Down)”的选股复盘逻辑（大盘环境 ➔ 行业风口 ➔ 滞涨龙头）。

### 1.1 交互式大盘 K 线与“时间穿梭机”
- **UI 布局**：占据 Tab 1 顶部约 40% 视口高度。
- **核心功能**：渲染沪深 300（`000300.SH`）日 K 线图，支持平移、滚轮缩放。
- **关键交互**：
  - 用户点击图表上的**任意一根 K 线（某一日）**时，全局状态 `selectedDate` 更新。
  - 图表侧边栏联动展示该日的**大盘量化诊断卡片**：
    - 大盘趋势判定（MA20/MA60 多空状态）。
    - 市场情绪指标（涨跌停家数比）。
    - 策略总控建议（允许开仓 / 冰点熔断）。

### 1.2 动态热点板块追踪 (Sector Momentum)
- **UI 布局**：位于大盘 K 线下方，横向滑动卡片 (Carousel) 或流式网格 (Grid)。
- **数据联动**：严格跟随 `selectedDate` 展示该日后台识别出的**主线/热点行业**。
- **展示字段**：行业名称、5日累计涨幅（标红凸显 >3%）、10日累计涨幅、量能扩张倍数。

### 1.3 活跃个股量化龙虎榜 (Active Stocks Ranking)
- **UI 布局**：Tab 1 底部的全宽数据表格 (`DataTable`)。
- **数据联动**：展示基于 `selectedDate` 和热点板块筛选出的**“滞涨首阳激活股”**。
- **展示列**：股票代码、名称、所属行业、个股5日涨幅（对比行业展现相对弱势）、当日量比（突出 >1.5倍）、综合量化评分。
- **关键交互**：**双击表格任意一行**，更新全局 `selectedStock`，并**自动路由跳转至 Tab 2**。

## 🔬 模块二：Tab 2 - 个股量化深度解析

**设计理念**：保留现有的专业极客风格，但图表和数据必须完全适配 v3.0 业务逻辑（真实摩擦成本、非对称盈亏比）。

### 2.1 注入灵魂的 K 线主图 (B/S 交易点位渲染)
- **主图渲染**：蜡烛图 + 均线系统（MA5, MA20, MA60）。
- **信号标记 (核心难点)**：
  - 读取后台传入的历史买卖记录，精准绘制 **买点（B - 红色向上箭头）** 与 **卖点（S - 绿色/蓝色向下箭头）**。
  - **Hover 交互**：鼠标悬停在 B/S 图标上时，弹出 `HoverCard` 或 `Tooltip` 显示交易明细（入场价、离场价、扣除 0.7% 摩擦成本后的净 PnL、出场归因）。
- **副图联动**：
  - **副图 1 (量能)**：成交量 + VMA20，高亮标注量比异常放大的“激活柱”。
  - **副图 2 (相对强度)**：绘制个股与所属行业的相对收益率走势线 (Relative Strength)。

### 2.2 量化因子与模型体检单
- **UI 布局**：K 线图右侧或下方的独立侧边栏。
- **展示内容**：
  - **微观结构**：当前 RSI/J 值、布林带分位、距离前高百分比。
  - **OOS 回测表现**：展示这套短线参数在该股上的基准数据（训练集/验证集胜率、盈亏比 PF、平均持仓周期）。

### 2.3 交易账本与归因分析 (Trade Ledger)
- **UI 布局**：Tab 2 底部的详细表格 (`DataTable`)。
- **数据展示**：该股历史所有模拟交易的逐笔明细。
- **视觉要求**：
  - 盈利行背景泛红（浅色），亏损行背景泛绿（浅色）。
  - **出场归因列**：必须使用 `shadcn/ui` 的 `<Badge>` 组件分类高亮出场原因（如：`止盈 (+6%)`、`止损 (-3%)`、`超时强制平仓 (5天)`、`追踪止盈`）。

## 📅 分阶段交付建议 (Action Plan)

为了控制风险并保证交互流畅度，建议按以下顺序开发：

1. **Phase 1 (骨架与状态)**：清理现有多余路由，搭建 Tab 1 和 Tab 2 基础框架，实现 `React Context` 全局日期和个股代码的联动。
2. **Phase 2 (Tab 1 交互)**：实现大盘图表点击事件与下方“热点板块”+“龙虎榜表格”的数据流打通。测试双击跳转逻辑。
3. **Phase 3 (K线图攻坚)**：集中精力解决 Tab 2 中 `lightweight-charts` 的 B/S Marker 渲染与 Tooltip 悬浮展示。
4. **Phase 4 (UI 打磨)**：全面引入 `shadcn/ui` 规范，打磨 Card 边距、暗黑模式适配与数据表格的条件格式渲染。
